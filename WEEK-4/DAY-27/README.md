## 🔎 Searching Algorithms Reviewed

### Linear Search

Revisited the basic approach of sequentially checking each element in an array until the target element is found or the array ends.

**Key Points Reviewed**

* Works on both sorted and unsorted arrays
* Simple implementation
* Time Complexity: **O(n)**

---

### Binary Search

Reviewed the divide-and-conquer strategy used to efficiently search elements in **sorted arrays**.

**Key Points Reviewed**

* Iterative and recursive implementations
* Mid calculation and boundary handling
* Time Complexity: **O(log n)**

---

### Binary Search Variations

Revisited common variations frequently asked in coding interviews.

**Topics Reviewed**

* First occurrence of an element
* Last occurrence of an element
* Count of occurrences

## 🔄 Sorting Algorithms Reviewed

### Bubble Sort

Revisited the concept of repeatedly swapping adjacent elements until the array becomes sorted.

**Key Points**

* Simple but inefficient for large datasets
* Best Case: **O(n)** (with optimization)
* Average/Worst Case: **O(n²)**

---

### Selection Sort

Reviewed the method of repeatedly selecting the **minimum element** from the unsorted portion and placing it at the correct position.

**Key Points**

* Always performs **O(n²)** comparisons
* Fewer swaps compared to Bubble Sort

---

### Insertion Sort

Revisited how elements are inserted into their correct position within a **growing sorted portion** of the array.

**Key Points**

* Efficient for **small or nearly sorted arrays**
* Best Case: **O(n)**
* Worst Case: **O(n²)**

---

## ⚡ Advanced Sorting Concepts Reviewed

### Merge Sort

Revisited the **divide-and-conquer approach** where the array is split recursively and merged in sorted order.

**Key Points**

* Stable sorting algorithm
* Time Complexity: **O(n log n)**
* Requires additional space for merging

---

### Quick Sort

Reviewed the concept of selecting a **pivot element** and partitioning the array around it.

**Key Points**

* Very efficient in practice
* Average Time Complexity: **O(n log n)**
* Worst Case: **O(n²)** if pivot selection is poor
